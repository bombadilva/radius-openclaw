# x402 Protocol Bridge

## What is x402?

x402 is a protocol for HTTP-native micropayments using the `402 Payment Required` status code. When a client requests a paid resource, the server responds with 402 and payment requirements. The client pays on-chain and retries with proof of payment.

## Radius x402 Flow

```
Client                    Server                   Radius
  |                         |                        |
  |  GET /api/resource      |                        |
  |------------------------>|                        |
  |                         |                        |
  |  402 + PaymentRequired  |                        |
  |<------------------------|                        |
  |                         |                        |
  |  Send payment --------->|----------------------->|
  |                         |                        |
  |  GET + X-PAYMENT header |                        |
  |------------------------>|                        |
  |                         |  Verify TX ----------->|
  |  200 + Resource         |<-----------------------|
  |<------------------------|                        |
```

## Server-Side: Express Middleware

```typescript
import { x402Middleware } from '@bombadilva/x402-bridge';

const app = express();

app.get('/api/premium-data',
  x402Middleware({
    payTo: '0xYourAddress...',
    amount: '0.01',
    asset: 'USD',
    network: 'testnet',
    description: 'Premium data access',
  }),
  (req, res) => {
    // This handler only runs after payment is verified
    res.json({ data: 'premium content' });
  },
);
```

### 402 Response Format

```json
{
  "error": "Payment Required",
  "accepts": {
    "scheme": "radius",
    "network": "testnet",
    "payTo": "0x1234...",
    "maxAmountRequired": "0.01",
    "asset": "USD",
    "resource": "/api/premium-data",
    "description": "Premium data access"
  }
}
```

## Client-Side: Auto-Payment

```typescript
import { x402Client } from '@bombadilva/x402-bridge';
import { createAgentWallet } from '@bombadilva/agent-wallet';

const kit = createAgentWallet({ ... });
const client = x402Client(kit, 0, { maxAmount: '1.00' });

// Automatically handles 402 responses
const response = await client.fetch('https://api.example.com/premium-data');
// If 402 → pays → retries → returns the actual response
```

## Payment Verification

The verifier checks:
1. Transaction exists on Radius
2. Correct amount was sent
3. Correct recipient received the funds
4. Transaction is confirmed (not pending)

```typescript
import { createVerifier } from '@bombadilva/x402-bridge';

const verifier = createVerifier(chainClient);
const result = await verifier.verify({
  txHash: '0xabc...',
  chainId: 72344,
  payer: '0xClient...',
}, {
  payTo: '0xServer...',
  amount: '0.01',
  asset: 'USD',
});

if (result.valid) {
  // Payment confirmed
}
```

## Comparison with Coinbase x402

| Feature | Coinbase x402 (Base) | Radius x402 |
|---------|---------------------|-------------|
| Network | Base L2 | Radius |
| Cost/tx | ~$0.001 | ~$0.0000001 |
| Finality | ~2s | <1s |
| Currency | USDC (ERC-20) | Native USD |
| Agent integration | Manual | OpenClaw plugin |
| Middleware | Express only | Express (more coming) |

## Configuration

### Environment Variables

```bash
# Verifier service (optional, for remote verification)
VERIFIER_URL=http://localhost:3003/verify

# Client settings
X402_MAX_AMOUNT=1.00        # Max auto-payment per request
X402_NETWORK=testnet        # Network to use
```
